Домашнее задание к уроку "Контроль версий Git"

Необзодимо выполнить следующее:

  • Создать новый проект и подключить в него Git через терминал.
  • Указать, что в вашем проекте будут игнорироваться все файлы с расширением .sass и папки: bin, admin, config.
  • Добавить все файлы в локальное хранилище;
  • Создать новую ветку и перейти в неё;
  • В новой ветке создайть папку blog с файлами: index.js, index.html;
  • Загрузить на удаленный репозиторий лишь основную ветку проекта.

Для отправки на проверку, необходимо создать файл, в котором будут прописанны все команды что необходимо было бы сделать для выполнения действий задания.

Выполнение задания

1. Создаем новый проект, например "homework" и подключаем в нем Git через терминал.

            # Инициализация git
            …\homework> git init
            # Установка имени пользователя
            …\homework> git config --global user.name “Victor”
            # Установка email пользователя
            …\homework> git config --global user.email  “victor@victor.com”

2. Создаем в проекте необходимые папки и файлы:

3. Содержание файла README.md (этот файл можно создать позже)

# Homework for the lesson "Git"


## The following must be done:

1. Create a new project and connect Git to it via the terminal.

2. Specify that all files with the .sass extension and folders: bin, admin, config will be ignored in the project.

3. Add all files to local storage;

4. Create a new branch and move to it;

5. In a new branch, create a blog folder with the files: index.js, index.html;

6. Upload only the main branch of your project to the remote repository.

4. Содержание файла .gitignore

*.sass

bin/

admin/

config/

5. Добавляем все файлы в локальное хранилище.

            # добавляем изменения в рабочем каталоге в индекс
            …\homework> git add
            # при необходимости проверяем статус файлов:
            …\homework> git status
            # добавляем все файлы из индекса в локальное хранилище указывая комментарий:
            …\homework> git commit -m “first version”

6. Создаем новую ветвь, переходим в нее.

            # создаем новую ветвь «blog»
            …\homework> git branch blog
            # переходим в ветвь «blog»
            …\homework> git checkout blog
            # при необходимости проверяем переход к новой ветке через
            …\homework> git status

7. Создаем папку с файлами для новой ветви и синхронизируем её


            # добавляем изменения в рабочем каталоге в индекс
            …\homework> git add
            # при необходимости проверяем статус файлов:
            …\homework> git status
            # добавляем все файлы из индекса в локальное хранилище указывая комментарий:
            …\homework> git commit -m “folder blog added”

8. Создаем новый репозиторий на портале github

9. Загружаем на удаленный репозиторий лишь основную ветку проекта.

            # переходим в основную ветку:
            …\homework> git checkout master
            # при необходимости проверяем переход к новой ветке через:
            …\homework> git status
            # подключаемся к удаленному репозиторию используя ссылку с портала github:
            …\homework> git remote add origin https://github.com/fvm80/homework.git
            # при необходимости проверяем подключение к удаленному репозиторию:
            …\homework> git remote
            # загружаем файлы из основной ветки на Github:
            …\homework> git push -u origin master
